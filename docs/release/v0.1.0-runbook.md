# v0.1.0 Release Runbook

This runbook is the P18 operator path for cutting CORTEX v0.1.0. It assumes a
clean release branch and an operator who has permission to push tags, publish
container images, sign with cosign, and create GitHub Releases.

## 1. Preflight

1. Confirm the working tree is clean.
2. Confirm all intended P0..P17 changes are merged into the release branch.
3. Confirm `CHANGELOG.md`, `README.md`, `docs/ARCHITECTURE.md`, and
   `docs/adr/INDEX.md` reflect the release boundary.
4. Confirm no production secrets are present in `infra/docker`,
   `infra/helm`, `infra/terraform`, `infra/ansible`, or Postman env files.
5. Confirm the current runtime broker statement is understood: Azure Service
   Bus is provisioned as the intended production target, but the application
   runtime remains Kafka-wired for v0.1.0.

## 2. Local Gates

Run the release-prep dry run:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass `
    -File scripts\live-e2e\smoke-p18-release-prep.ps1
```

Run the phase gates that are relevant to the release branch:

```powershell
.\mvnw.cmd verify -B
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\live-e2e\smoke-p10.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\live-e2e\smoke-p11-helm.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\live-e2e\smoke-p12-terraform.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\live-e2e\smoke-p13-ansible.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\live-e2e\smoke-p17-grafana.ps1
```

Run Newman against the same live JVM/compose generations required by the
phase-specific runbooks.

## 3. Generate SBOM

```powershell
powershell -NoProfile -ExecutionPolicy Bypass `
    -File scripts\release\generate-sbom.ps1 -Version 0.1.0
```

Expected outputs:

- `target/cyclonedx-sbom.json`
- `target/cyclonedx-sbom.xml`

## 4. Build and Push Images

Build and push the eight runtime images from `infra/docker`. The image tags
must use the release version, for example:

```text
ghcr.io/varadharajaan/cortex/log-gateway:0.1.0
```

P14 will eventually automate this. For v0.1.0, keep the image build log with
the release evidence.

## 5. Sign Images

Dry-run first:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass `
    -File scripts\release\sign-images.ps1 -Version 0.1.0 -DryRun
```

Actual signing requires explicit confirmation:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass `
    -File scripts\release\sign-images.ps1 -Version 0.1.0 -ConfirmSign SIGN
```

## 6. Create Tag and GitHub Release

Dry-run first:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass `
    -File scripts\release\create-github-release.ps1 -Version 0.1.0 -DryRun
```

Actual publication requires explicit confirmation:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass `
    -File scripts\release\create-github-release.ps1 `
    -Version 0.1.0 `
    -ConfirmRelease RELEASE
```

## 7. Rollback

1. Do not delete the Git tag after public publication unless the release is
   proven unrecoverable.
2. Mark the GitHub Release as pre-release or withdrawn if the artifact must be
   hidden from operators.
3. Revert image tags by updating deployment values to the previous working tag.
4. Use `infra/ansible/playbooks/rollback.yml` for Helm rollback when the
   release reached a Kubernetes cluster.

## 8. Evidence to Archive

- Maven verify log.
- P10/P11/P12/P13/P17 smoke logs.
- Newman logs.
- SBOM JSON/XML.
- Cosign verification output for every image.
- GitHub Release URL.
- Final `git rev-parse HEAD` and `git tag -v v0.1.0` output if signed tags are
  enabled locally.
